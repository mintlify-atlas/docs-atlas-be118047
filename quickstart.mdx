This guide will help you integrate the Mailee.me .NET SDK into your application and create your first contact.

## Prerequisites

- .NET Framework 4.0 or higher
- A Mailee.me account with API access
- Your API credentials (Secret Key and Subdomain)

<Steps>

<Step title="Add the SDK to your project">

Download the Mailee.me .NET SDK DLL and add it as a reference to your project:

1. Right-click on your project in Solution Explorer
2. Select **Add Reference**
3. Browse to the Mailee.dll file and add it
4. Ensure the reference appears in your project's References folder

Alternatively, if you're building from source, add the Mailee.Src project to your solution and reference it.

</Step>

<Step title="Configure your Web.Config">

Add your API credentials to the `<appSettings>` section of your Web.Config file:

```xml
<configuration>
  <appSettings>
    <!-- Mailee - Secret Key -->
    <add key="MaileeKey" value="cb3n376h66a1r"/>
    <!-- Mailee - Subdomain -->
    <add key="MaileeSubdomain" value="seusite"/>
  </appSettings>
</configuration>
```

<Info>
You can find your API credentials in your Mailee.me dashboard. The Secret Key and Subdomain together form your API URL: `http://api.{SecretKey}.{Subdomain}.mailee.me/`
</Info>

</Step>

<Step title="Import the namespace">

Add the Mailee namespace to your C# file:

```csharp
using Mailee;
```

</Step>

<Step title="Create your first contact">

Now you're ready to create a contact in Mailee.me:

<CodeGroup>

```csharp Basic Example
try
{
    // Create a new contact
    Mailee.Contact contact = new Mailee.Contact();
    contact.Email = "john.doe@example.com";
    contact.Name = "John Doe";
    contact.Company = "Acme Corporation";
    contact.Phone = "(555) 123-4567";

    // Save the contact
    contact.save();

    Console.WriteLine("Contact created successfully!");
}
catch (Mailee.RestExcepction ex)
{
    // Handle API errors
    foreach (string error in ex.ErrorMessages)
    {
        Console.WriteLine("Error: " + error);
    }
    Console.WriteLine("Status Code: " + ex.StatusCode);
    Console.WriteLine("Status Description: " + ex.StatusDesciption);
}
catch (Exception ex)
{
    // Handle general errors
    Console.WriteLine("Exception: " + ex.Message);
}
```

```csharp Complete Example
try

    // Handle general errors

```

```csharp ASP.NET Example
protected void CreateContact_Click(object sender, EventArgs e)

    {

            errorMessage += "- " + error + "<br/>";

        lblMessage.Text = errorMessage;
        lblMessage.CssClass = "error";
    }
    catch (Exception ex)
    {
        lblMessage.Text = "An unexpected error occurred: " + ex.Message;
        lblMessage.CssClass = "error";
    }
}
```

## Understanding Error Handling

The SDK uses a custom exception class `Mailee.RestExcepction` for API errors. This exception provides:

- **ErrorMessages**: A list of error messages returned by the API
- **StatusCode**: The HTTP status code (e.g., 400, 401, 500)
- **StatusDesciption**: The HTTP status description

<Warning>
Always wrap your API calls in try-catch blocks to handle potential errors gracefully. Network issues, invalid credentials, or validation errors can cause exceptions.
</Warning>

## Common Issues

<Info>
**Configuration not found**: If you get an error about missing configuration, verify that your Web.Config has the `MaileeKey` and `MaileeSubdomain` settings in the `<appSettings>` section.
</Info>

<Info>
**401 Unauthorized**: Check that your Secret Key and Subdomain are correct. The API URL format must be: `http://api.{SecretKey}.{Subdomain}.mailee.me/`
</Info>

<Info>
**Contact not created**: The API returns the created contact's ID after successful creation. If `contact.Id` is 0 after calling `save()`, check for validation errors in the exception handler.
</Info>
